The monthly amounts paid into each pension. Where a member's hours vary, contributions can differ by working-days tier (5-, 4- or 3-day week) and the model picks the row matching their current phase — so each tier is shown clearly. Each row is saved separately; nothing is stored until you tap its Save, and Reload discards edits.

August increase is entered as a friendly percentage (e.g. 3 for 3%) — the model applies it once each August to grow the contribution over time.

Contribution Exceptions

Adjust contributions for a period without changing the standard rows above. Each exception has a member, pension, date range and value.

One-off (ticked): a lump sum (e.g. a bonus) added on top of the normal contribution, applied once at the start date. Override (unticked): replaces the normal monthly contribution with this value for every month from start to end.
